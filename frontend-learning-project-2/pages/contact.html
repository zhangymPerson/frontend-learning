体验 TypeScript 类型安全的表单处理

📝 联系表单

🔧 TypeScript 如何让表单更安全

Step 1：定义接口（在 ts/types.ts 中）

// 定义表单数据的"形状"
interface ContactFormData {
  name: string;     // 必须是字符串
  email: string;    // 必须是字符串
  message: string;  // 必须是字符串
}

// 定义验证结果的"形状"
interface ValidationResult {
  isValid: boolean;
  errorMessage: string;
}

Step 2：获取表单数据（类型安全）

// 断言元素类型，TS 知道 input 有 .value 属性
const nameInput = document.getElementById('nameInput') as HTMLInputElement;
const name: string = nameInput.value.trim();  // 返回 string

// 返回值必须符合 ContactFormData 接口
const getFormData = (): ContactFormData => {
  return { name, email, message };  // 少一个属性？编译报错！
};

Step 3：验证（返回类型约束）

// 参数类型 + 返回类型都有标注
const validateForm = (data: ContactFormData): ValidationResult => {
  // 解构赋值，类型自动继承
  const { name, email, message } = data;

  if (!name || !email || !message) {
    return { isValid: false, errorMessage: '请填写所有字段！' };
  }
  // ...
};

🔑 核心价值：每个函数的输入和输出都有明确的类型约束。 传错参数类型、返回值形状不对 → 编译器立刻报错，不会等到运行时才发现 bug！
