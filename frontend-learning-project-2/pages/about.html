📖 TypeScript 学习指南

理解 TypeScript 的核心概念和工作流程

⚙️ TypeScript 工作流程

  你写的代码            编译器处理           浏览器运行
  ─────────────        ─────────────        ─────────────
  ts/main.ts    →→→    tsc 编译      →→→    js/main.js
  (TypeScript)         (类型检查)           (纯 JavaScript)
  (带类型标注)         (去掉类型)           (浏览器能直接运行)
  (你维护的源码)       (检查类型错误)        (用户看到的代码)

💡 关键理解：TypeScript 不是浏览器能直接运行的语言。 它需要先"编译"（也叫"转译"）成 JavaScript。浏览器最终执行的是编译后的 JS。 TypeScript 的价值在于开发阶段帮你发现错误。

💡 TypeScript 核心概念

🏷️ 类型标注

let name: string = "你好"

给变量声明类型，写错了编辑器立刻报错。TypeScript 最基础也最重要的能力。

📋 接口 Interface

interface User { name: string; age: number }

定义对象的"形状"。对象必须符合接口的约束，不能多也不能少。

🔀 联合类型

let val: string | number

一个变量可以是多种类型之一。搭配"类型收窄"使用，先判断再操作。

🎯 类型断言

el as HTMLInputElement

当 TypeScript 推断的类型不够精确时，你告诉它"我知道更准确的类型"。

📊 枚举 Enum

enum Color { Red, Green, Blue }

定义一组命名常量。避免到处写魔法字符串，写错了编译器会告诉你。

🧠 类型推断

const x = 42 // TS 知道 x 是 number

TypeScript 很聪明，大部分时候能自动推断类型。你不需要每个都手动标注。

🔄 JavaScript → TypeScript：代码对比

JavaScript 写法（项目1）：

// 项目1：JavaScript
let clickCount = 0;

function sayHello() {
  const el = document.getElementById('greetingText');
  el.textContent = '你好！';  // 如果 el 是 null，运行时才崩溃！
}

TypeScript 写法（项目2）：

// 项目2：TypeScript
let clickCount: number = 0;  // 类型标注：这是数字

const sayHello = (): void => {
  const el = document.getElementById('greetingText');

  if (!el) return;  // 空值检查：编译器要求你处理 null 的情况

  el.textContent = '你好！';  // 这里 TypeScript 知道 el 不是 null，安全！
};

🔑 关键区别：TypeScript 在编译阶段就发现了 el 可能是 null 的问题， 强制你处理这种情况。JavaScript 要等到运行时才会崩溃。
